# How tools get picked

No fancy methodology here. Roughly:

1. **Is anyone actually using it?** — Downloads, GitHub stars, community activity, people mentioning it in real conversations.
2. **Does it do something different?** — If it's a clone of something already listed with no clear advantage, it probably doesn't need to be here.
3. **Is it maintained?** — Dead projects get moved to honorable mentions or removed.
4. **Can you actually try it?** — Preference for things with real docs, a working install, and a path to getting started.

When in doubt, we lean toward including things that are genuinely useful over things that are just popular, and things that are popular over things that are just interesting.

If you think something's ranked wrong or missing, open a PR.
