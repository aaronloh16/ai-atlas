# Autonomous Agents

Autonomous or semi-autonomous systems that execute multi-step software tasks with minimal supervision.

## Top tools

## OpenHands
- Link: https://github.com/OpenHands/OpenHands
- Category: Autonomous Agents
- What it is: AI-driven software development agent capable of end-to-end coding tasks.
- Why it matters: One of the clearest open-source anchors for autonomous software engineering.
- Best for: Teams experimenting with autonomous bugfixing and feature delivery.
- Best paired with: OpenHands Software Agent SDK, MCP, eval tooling.

## OpenHands Software Agent SDK
- Link: https://github.com/OpenHands/software-agent-sdk
- Category: Autonomous Agents
- What it is: SDK for building and extending software agents in OpenHands workflows.
- Why it matters: Enables custom autonomous coding behaviors and integrations.
- Best for: Builders creating specialized engineering agents.
- Best paired with: OpenHands, Langfuse, Promptfoo.

## Trae Agent
- Link: https://github.com/bytedance/trae-agent
- Category: Autonomous Agents
- What it is: Open-source agent project focused on software and task automation workflows.
- Why it matters: Adds diversity in architecture and execution models for autonomous dev.
- Best for: Builders comparing emerging autonomous coding approaches.
- Best paired with: MCP, observability stack.

## OpenClaw
- Link: https://github.com/openclaw/openclaw
- Category: Autonomous Agents
- What it is: Personal AI agent platform with workflow and automation orientation.
- Why it matters: Demonstrates productized autonomous agent patterns beyond simple chat.
- Best for: Teams building assistant-like autonomous products.
- Best paired with: Memory systems, MCP servers.

## Honorable mentions

## Cline
- Link: https://github.com/cline/cline
- Category: Autonomous Agents
- What it is: IDE-based coding agent with iterative task execution.
- Why it matters: Useful midpoint between autocomplete and full autonomy.
- Best for: Developers wanting controlled autonomy inside editor.
- Best paired with: Promptfoo, Langfuse.

## How to choose

- Use `OpenHands` as your default baseline for autonomous software development.
- Add SDK-driven extension when you need domain-specific behaviors.
- Treat eval/observability as mandatory before increasing autonomy.
