# AI Coding Tools

Tools that help you write, modify, and ship code with AI.

## Top tools

## Claude Code
- Link: https://docs.anthropic.com/en/docs/claude-code/overview
- Category: AI Coding Tools
- What it is: Anthropic's terminal-first coding agent.
- Why it matters: Highest CLI download numbers right now (~35M npm downloads/month). Strong autonomous repo operations.
- Best for: Developers who ship from terminal.
- Best paired with: Promptfoo, Langfuse, MCP, GitHub Actions.

## GitHub Copilot
- Link: https://marketplace.visualstudio.com/items?itemName=GitHub.copilot
- Category: AI Coding Tools
- What it is: IDE-native coding assistant from GitHub/Microsoft.
- Why it matters: Massive VS Code install base (72M+). The default for many enterprise teams.
- Best for: Teams with IDE-first workflows and existing GitHub footprint.
- Best paired with: Copilot Chat, Promptfoo, Langfuse.

## Cursor
- Link: https://cursor.com/
- Category: AI Coding Tools
- What it is: AI-first IDE with integrated agent workflows.
- Why it matters: Strong developer mindshare and frequent capability releases.
- Best for: Developers who prefer integrated IDE agent loops.
- Best paired with: Promptfoo, Langfuse, MCP.

## OpenHands
- Link: https://github.com/OpenHands/OpenHands
- Category: AI Coding Tools
- What it is: Open-source autonomous software engineering agent.
- Why it matters: 68K+ GitHub stars, very active development. Higher autonomy than typical coding assistants.
- Best for: Teams evaluating autonomous coding loops.
- Best paired with: Langfuse, Promptfoo.

## Aider
- Link: https://github.com/paul-gauthier/aider
- Category: AI Coding Tools
- What it is: Open-source terminal AI pair programmer with deep Git integration. Works with 100+ languages and multiple LLMs.
- Why it matters: 25K+ stars. Unique Git-native workflow with automatic commits, multi-file editing. Not locked to one provider.
- Best for: Developers who want open-source, provider-agnostic CLI coding.
- Best paired with: Ollama (local models), Claude/GPT APIs, Git workflows.

## App builders (vibe coding)

## Bolt.new
- Link: https://bolt.new
- Category: AI Coding Tools
- What it is: Browser-based AI app builder running Node.js via WebContainers. Supports React, Vue, Svelte, Next.js, Remix.
- Why it matters: Fastest path from prompt to running app with no local setup. Broadest framework support.
- Best for: Rapid prototyping and testing ideas before committing to a full codebase.
- Best paired with: Cursor (for export and iteration), Vercel (for deployment).

## Lovable
- Link: https://lovable.dev
- Category: AI Coding Tools
- What it is: AI fullstack engineer (formerly GPT Engineer) with deep Supabase integration for database, auth, and row-level security.
- Why it matters: Most accessible path from idea to full-stack app for non-technical builders.
- Best for: Non-technical founders who need working apps with real backends.
- Best paired with: Supabase, Bolt.new, deployment platforms.

## v0
- Link: https://v0.dev
- Category: AI Coding Tools
- What it is: Vercel's AI UI generator — produces React components from natural language.
- Why it matters: Best-in-class UI quality. Exports standard React code with minimal lock-in.
- Best for: Frontend devs and designers who want high-quality components fast.
- Best paired with: Vercel AI SDK, Next.js, shadcn/ui.

## Replit Agent
- Link: https://replit.com/ai
- Category: AI Coding Tools
- What it is: Browser-based autonomous coding agent with 200-minute runtime — builds, tests, and fixes apps.
- Why it matters: Zero-setup path to working software. Can build mobile apps and create other agents.
- Best for: Beginners, rapid prototyping, browser-first development.
- Best paired with: Deployment platforms, Slack/Notion integrations.

## How to choose

**Coding assistants:**
- Choose `Claude Code` for terminal-first velocity.
- Choose `GitHub Copilot` for mainstream IDE adoption and enterprise familiarity.
- Choose `Cursor` for an AI-native IDE with strong agent loops.
- Choose `Aider` for open-source, provider-agnostic CLI coding with Git-native workflow.
- Choose `Gemini CLI` for free, high-context CLI coding.
- Choose `Amazon Q Developer` when building on AWS.
- Choose `Continue` when provider control and customization matter most.
- Choose `OpenHands` when you want more autonomy and can enforce review gates.

**App builders:**
- Choose `Bolt.new` for fastest prototyping with broadest framework support.
- Choose `Lovable` when you need full-stack with database and auth.
- Choose `v0` when UI quality and React generation is the priority.
- Choose `Replit Agent` for zero-setup browser-first building.
